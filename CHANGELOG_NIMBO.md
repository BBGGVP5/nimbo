## Обновление от 27 мая 2026

- Доработана страница "Маршрутизация":
  - кнопка "Добавить свой" теперь открывает редактор нового routing-профиля;
  - шестерёнка профиля заменена на рабочее редактирование;
  - в редакторе можно менять название, описание, стратегию доменов, порядок правил, DNS-поля, флаги global proxy / bypass private IP / FakeDNS и списки direct/proxy/block правил;
  - пользовательские профили можно удалить с подтверждением;
  - встроенные профили по-прежнему защищены от удаления.

  - fallback User-Agent и данные обновлений в предпросмотре используют актуальную версию;
  - `package-lock.json` для UI и installer синхронизированы с `1.0.0`.

- Исправлен текст процесса в установщике:
  - статус текущего шага больше не показывает обрубленное `Идет`;
  - при установке отображается `Установка`, при удалении - `Удаление`;
  - общий статус процесса заменён на `Установка Nimbo` / `Удаление Nimbo`.

- Проверки:
  - `npm run build` для `apps/ui`;
  - `npm run build` для `apps/installer`;
  - проверка в браузерном предпросмотре: создание, сохранение, удаление routing-профиля и отображение версии в "О программе".

## Обновление от 20 мая 2026

- Исправлено поведение окна конфликтующего ПО:
  - после выгрузки процессов выполняется повторная проверка реального состояния;
  - если процесс уже исчез, приложение больше не должно снова просить выгрузить его на следующем запуске;
  - Cloudflare WARP service (`warp-svc.exe` / `warpsvc`) исключен из конфликтов, чтобы не показывать повторяющееся предупреждение для службы, которую нельзя корректно выгрузить обычным способом;
  - подсказки в окне конфликтов больше не упоминают Cloudflare WARP как рекомендуемый конфликт.

- Улучшен блок серверов:
  - свернутая панель серверов больше не растягивается пустым высоким контейнером;
  - свернутое состояние оформлено как компактный док, чтобы кнопки не болтались отдельно от интерфейса.

- Доработаны состояния кнопки подключения:
  - убрано зеленое свечение при наведении, когда VPN уже подключен;
  - в светлой теме кнопка подключения и текст "Нажмите для подключения" стали мягче и меньше выбиваются из общего интерфейса.

- Обновлен установщик:
  - у кнопки "Закрыть" убрана иконка;
  - кнопки "Закрыть" и "Открыть Nimbo" выровнены по высоте;
  - кнопка "Открыть Nimbo" сделана длиннее и заметнее на финальном экране.

- Акцентный цвет теперь сильнее влияет на материалы интерфейса:
  - боковая панель, панели, карточки, список серверов и другие glass/material элементы берут оттенок из выбранного акцента;
  - светлая и темная темы используют общие accent-aware токены для бордеров, фонов, бликов и теней.

- Добавлена чёрная OLED-тема:
  - в настройках внешнего вида появился отдельный режим "Чёрная";
  - фон становится натурально чёрным;
  - блоки, панели и стеклянные материалы становятся темнее и мягче, сохраняя выбранный акцентный цвет.

- Добавлен кастомный Linux-установщик:
  - новый скрипт `apps/installer/build-custom-linux-installer.sh` собирает Linux payload и self-contained custom installer binary;
  - Linux-ветка установщика вшивает `nimbo-ui` без `.exe`, ставит бинарник `nimbo`, `icon.png`, `.desktop` entry и регистрирует `nimbo://` через `xdg-mime`, если утилита доступна;
  - Windows-артефакты custom installer теперь складываются в `target/release/bundle/custom/windows`;
  - Linux-артефакты custom installer складываются в `target/release/bundle/custom/linux`.

- Материалы сделаны прозрачнее и ближе к glassmorphism:
  - снижена плотность material/glass фонов в светлой и темной теме;
  - ослаблены тени и бордеры;
  - усилены blur и saturation у основных панелей, сайдбара, карточек и панели серверов.

- Проверки:

## Что было проделано

- Исправлен TUN/Xray runtime:
  - включены `quic` и `fakedns` в sniffing;
  - `routeOnly` оставлен `false`, чтобы Xray мог корректно восстанавливать домен из FakeDNS;
  - некорректный `dns.queryStrategy: IPIfNonMatch` в шаблонах нормализуется в `UseIP`;
  - для `freedom` outbound добавляется `domainStrategy: UseIP`, чтобы direct-трафик резолвился через Xray DNS, а не через чужой FakeDNS;
  - FakeDNS-адреса `198.18.0.0/15` больше не используются как реальные bypass IP;
  - добавлен DoH fallback для получения реального IPv4 сервера;
  - при настройке и очистке TUN дополнительно очищается DNS cache Windows.

- Доработана работа с Xray JSON-шаблонами:
  - поддержан выбор шаблона по `xrayJsonTemplateUuid` / `xray_json_template_uuid`;
  - шаблоны подтягиваются из подписки и публичных endpoints;
  - сохраняются routing rules, balancers и дополнительные proxy outbounds из шаблона;
  - full-config шаблоны санитизируются и дополняются runtime-настройками Nimbo.

- Улучшена логика direct/proxy маршрутизации:
  - direct outbound привязывается к физическому IPv4 через `sendThrough`, чтобы direct-сокеты не зацикливались обратно в TUN;
  - приватные IP и заданные direct-правила остаются выше общего proxy fallback.

- Добавлено окно конфликтующего ПО при запуске приложения:
  - Nimbo сканирует запущенные процессы при старте в TUN-режиме;
  - показывает модальное окно в стиле Nimbo;
  - можно выгрузить найденные процессы прямо из окна;
  - проверка больше не блокирует ручное подключение перед каждой попыткой.
  - после выгрузки выполняется повторный scan, а сырой PowerShell output больше не показывается в интерфейсе.
  - окно сделано компактнее, а сканер теперь ищет конфликты по имени процесса, пути и command line, включая Incy.

- Обновлено отображение серверов и профилей:
  - используются описания серверов из подписки, если они есть;
  - endpoint остаётся fallback-описанием;
  - отображаются transport/protocol badges, задержки, избранное и сортировки;
  - главная и страница профилей работают с выбранной активной подпиской и сервером.
